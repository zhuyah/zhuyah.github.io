Redis持久化方式简单对比

Posted on 2020-06-17
1.RDB
  • RDB是基于快照方式，每一次都是dump全量数据
  • 默认方式
  • Fork子进程进行快照写入，不影响父进程
  • 会丢失最后一次快照后的数据
  • 恢复快
  • save 60 5000 60秒内5000此写入触发快照，可以配置多个，任何一种满足都会触发
2.AOF
  • AOF，append only file，以追加方式记录日志
  • 记录所有写操作，文件体积大，恢复慢
  • always 每次写都同步写入磁盘，影响redis性能
  • everysec 每秒执行一次，显示将多个命令同步到磁盘，AOF默认策略
  • no 不进行持久化
